# Test Plan

**Project:** DemoQA Manual Testing Portfolio

| Field       | Details                   |
| ----------- | ------------------------- |
| Module      | Text Box                  |
| Version     | 1.0                       |
| Prepared By | Sravani Vanam             |
| Test Level  | Manual Functional Testing |

---

# 1. Purpose

Define the testing approach, scope, objectives, resources, risks, and deliverables for testing the DemoQA Text Box module.

---

# 2. Objectives

* Verify all functional requirements.
* Validate user input.
* Verify successful form submission.
* Identify functional defects.
* Produce complete QA documentation.

---

# 3. Scope

## In Scope

* Full Name
* Email
* Current Address
* Permanent Address
* Submit Button
* Displayed Output

## Out of Scope

* Performance Testing
* Security Testing
* Accessibility Testing
* API Testing

---

# 4. Test Strategy

The following testing types will be performed:

* Functional Testing
* Positive Testing
* Negative Testing
* Boundary Value Analysis
* Input Validation Testing
* UI Validation Testing

---

# 5. Test Environment

| Item    | Value           |
| ------- | --------------- |
| OS      | Windows 11      |
| Browser | Google Chrome   |
| AUT     | DemoQA Text Box |

---

# 6. Entry Criteria

* Requirements approved.
* Application accessible.
* Test Scenarios completed.
* Test Cases reviewed.

---

# 7. Exit Criteria

* All planned test cases executed.
* Critical defects documented.
* Test Summary Report prepared.

---

# 8. Deliverables

* SRS
* Test Plan
* Test Scenarios
* Test Cases
* RTM
* Test Execution Report
* Bug Reports
* Test Summary Report

---

# 9. Risks

| Risk                    | Mitigation                               |
| ----------------------- | ---------------------------------------- |
| Application unavailable | Resume testing when available            |
| Requirement changes     | Update impacted scenarios and test cases |

---

# 10. Roles

| Role        | Responsibility          |
| ----------- | ----------------------- |
| QA Engineer | Design, execute, report |
