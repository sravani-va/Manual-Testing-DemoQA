# Test Plan

## 1. Project Information

**Project Name:** DemoQA Manual Testing Portfolio

**Application Under Test (AUT):** DemoQA

**URL:** https://demoqa.com

**Prepared By:** Sravani Vanam

**Date:** July 2026

---

# 2. Purpose

The purpose of this test plan is to define the testing approach, scope, objectives, resources, schedule, and deliverables for the manual testing of selected DemoQA modules.

---

# 3. Objectives

- Verify that the application functions as expected.
- Identify functional defects.
- Ensure user input is processed correctly.
- Validate UI behaviour.
- Produce standard QA documentation.

---

# 4. Scope

## In Scope

- Text Box
- Check Box
- Radio Button
- Web Tables
- Buttons
- Links
- Upload & Download
- Dynamic Properties
- Practice Form
- Alerts
- Frames
- Browser Windows

## Out of Scope

- Performance Testing
- Security Testing
- Accessibility Testing
- Mobile Testing

---

# 5. Test Types

The following testing types will be performed:

- Functional Testing
- Positive Testing
- Negative Testing
- Boundary Value Testing
- UI Testing

---

# 6. Entry Criteria

Testing will begin when:

- Requirements are available.
- The application is accessible.
- Test environment is ready.

---

# 7. Exit Criteria

Testing will be completed when:

- All planned test cases have been executed.
- Critical defects have been reported.
- A Test Summary Report has been prepared.

---

# 8. Deliverables

The following documents will be produced:

- Software Requirements Specification (SRS)
- Test Plan
- Test Scenarios
- Test Cases
- Requirement Traceability Matrix (RTM)
- Bug Reports
- Test Execution Report
- Test Summary Report

---

# 9. Risks

Potential risks include:

- Incomplete requirements.
- Environment instability.
- Browser compatibility issues.
